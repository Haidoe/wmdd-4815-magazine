Getting Started With Vue

I have been exploring the front end for quite sometime now. I already have experiences working with jQuery, AngularJS, React, KnockoutJS as tools yet among them I find Vue most fascinating.

Vue is a progressive JavaScript framework created by Evan You.

It was made popular after Taylor Otwell, creator of Laravel, a popular PHP framework, tweeted about how convenient it is to use. Since then, Vue has been a default Frontend framework inside Laravel.

You could just say, Vue is a JavaScript UI library where it is being used building user interfaces in DATA DRIVEN way.

So, Why Vue?

Since 2014, millions of developer have chosen Vue because it has everything you need to quickly create and scale truly reactive products. Regardless of what scale is your product, wether you're working with Freelance Project, with new Startup or even Mature Enterprises company. Vue is a trusted frontend technology that gives you the right tool to do the job no matter how big the product you are building.

And among the JavaScript Frameworks or Libraries I stated above, Vue is the easiest to learn.


Before we start, let's review first the traditional way and data-driven way of updating the DOM.


Traditional Way

  1. Perform a query selector from the DOM.
  2. From the data you queried, apply your desired process.
  3. Then update the DOM Manually
  4. Repeat.

Data-Driven Way

  1. Inject the data property in the DOM
  2. Every time you updated the data property, it will automatically reflect in the DOM

Isn't Amazing?

Let's get started!

My goal in this article is to introduce to you Vue. We will create a simple app that accepts any input from user and display it in a list in a data-driven way.

First, we need to import Vue package inside our HTML file. You can just copy the codes below and paste inside the body. 

            
<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>

Then let's create a div with an id that has a value of "app".

You could put anything in the div's id, but for now, I will stick with app. You need to take note of the value you put inside the ID, we will use it to inject our data properties.

Then, let's add the following markups for our application:

  • Input element where we can get user's input
  • Button
  • Undordered List where we will display all user's input.

This is how it should look like:

            
<body>
  <script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
 
  <div id="app">
    <label>
      User Input
      <input type="text">
    </label>

    <button> Save </button>

    <!--This is where we put user's inputs -->
    <ul>
    </ul>
  </div>
</body>

We already prepared the DOM.


Now, Let's create a script tag.

Then, let's create an object with a method named data returning an object, and a property named methods.

The methods property will be an object where we will store our methods or functions. Please note that the names of the property must be follow (data and methods).


This is initially how it will look like:

          
const obj = {
  data() {
    return {}
  },
  methods: {}
}

These are the properties we need to inject to the DOM:

  • A state storing the list of User Inputs
  • A method for saving user's input to the list
  • This is optional: a state storing the current user's input

All the state we need must be part of the returned object inside the data method.

For this application, we have 2, a string for user's input and an array of user inputs. 

          
const obj = {
  data() {
    return {
      userInput: "",
      userInputList: []
    }
  },
  methods: {}
}

And we should have a function that will store user's input to the array list. 

          
const obj = {
  data() {
    return {
      userInput: "",
      userInputList: []
    }
  },
  methods: {
    handleSave() {
      //Adding the userInput to the list
      this.userInputList.push(this.userInput);

      //Will clear the value of the userInput
      this.userInput = "";
    }
  }
}

Now, the data properties we need is already completed. But we are only half done, we still need to use Vue directives to the DOM in order the reactivity to work.

What are Vue directives?

These are special HTML attributes that begins with v- that will allow Vue to manipulate the DOM in abstract.

For this app, we will be using 5 Vue Directives:

  • v-model will bind data property in two-way
  • v-on will listen to DOM events and run the JavaScript code after getting triggered.
  • v-if will display the HTML Node and it's children when the condition is true
  • v-for will allow you to render a list in more efficient way.
  • v-text will allow you to update element's text.

First, we need to bind the userInput property from data method to our input element using v-model. 

          
<input type="text" v-model="userInput">

Second, we need to add an event listener to the button using v-on and trigger handleSave from methods property 

          
<button v-on:click="handleSave"> Save </button>

Third, we need to use v-if inside the <ul> just to make sure it will only be rendered in the DOM if the userInputList has a value. 

          
<ul v-if="userInputList.length"> ... </ul>

For the last one, we will be using v-for with v-text. The v-for directive requires a special syntax in the form of item in items, where items is the source data array and item is an alias for the array element being iterated on. And we will just use v-text to update the <li>'s text 

          
<ul v-if="userInputList.length">
  <li v-for="item in userInputList">
    <span v-text="item"></span>
  </li>
</ul>

Finally, we already prepared everything.


For the last part, since we already imported the Vue package awhile ago, we already have an access with Vue Object globally where we will be using Vue's createApp method and then pass the object(obj) we created as an argument which returns an object with a mount property where we will pass the id of the element where we want to inject Vue's reactivity. 

          
//Passed the created object(obj) as the argument of Vue.createApp.
//Injecting the reactivity inside the div with an id of "app" (mount('#app'))
Vue.createApp(obj).mount('#app')
